基本概念：
连通分量：连通分量通常指的是一个图中最大的子图，其中任意两个顶点之间都有路径相连。（极大连通子图）
2 流形：2流形的网格就像用很多小纸片（多边形）粘成的 “超级纸”，每个地方局部看都是平的（或半平的，带边界），没有 “粘错” 导致的分叉、多层重叠等奇怪结构。
多边形汤（Polygon Soup）：“零散的多边形碎片”
多边形网格（Polygon Mesh）：“有序连接的多边形网络”


特征	多边形汤（Polygon Soup）	    多边形网格（Polygon Mesh）
边共享	 可能不共享、或被多个面共享（无效）	最多被 2 个面共享（边界边被 0 个，内部边被 2 个）
孤立元素	存在孤立顶点、边、面	      无孤立元素，所有元素构成连续网络
流形性质	可能是非流形（如三岔边、自交）	必须是 2 流形（局部像平面或半平面）
连通性	  无明确连通关系	             有明确的连通分量（如多个独立物体）
用途	 不可直接用于建模、渲染（需预处理）	可用于建模、渲染、物理模拟等


表面网格处理的完整流程，包含两个核心步骤：网格细化（三角剖分优化）和区域平滑（表面光顺），最终实现对表面补片的局部精细化和光滑化。
整体流程总结
输入：一个需要处理的表面补片（可能带有粗糙或不均匀的网格）。
细化步骤：
插入新顶点，加密边界附近的三角形（按 [9] 的密度准则）；
翻转无效边（满足两大条件），修复网格拓扑和几何质量，得到有效的三角剖分。
平滑步骤：
选定细化后的区域，通过优化双拉普拉斯系统（带边界约束），重新定位顶点，使表面光滑且切向连续。
输出：处理后的高质量表面补片。
核心思想
细化解决网格的 “局部密度和有效性” 问题（避免退化、保证三角剖分质量）；
平滑解决网格的 “全局光滑性” 问题（通过能量优化，使表面过渡自然）。
两者结合，实现从 “粗糙 / 不规则网格” 到 “精细 / 光滑表面” 的转换，常用于计算机图形学、几何建模或网格优化任务（如 3D 模型修复、细分曲面生成）。

API:
Meshing：
1. 网格处理的核心函数
(1) 细化函数 CGAL::Polygon_mesh_processing::refine()
功能：对网格上的任意区域（通过输入一组面 facets 指定）进行细化。
作用：提升目标区域的网格密度和质量（如加密三角形、优化连接方式），适用于局部精细化处理。
(2) 平滑函数 CGAL::Polygon_mesh_processing::fair()
功能：对网格上的任意区域（通过输入一组顶点 vertices 指定）进行平滑处理。
依赖：需要稀疏线性求解器，推荐使用 Eigen 3.2 或更高版本。
注意事项：若作为边界条件的固定顶点不足以求解构建的线性系统，平滑可能失败（需确保边界约束充分）。
2. 网格三角化需求
背景：许多算法要求网格的所有面具有相同的度数（如均为三角形），因此需要对多边形面（非三角形面）进行三角化。
实现函数：CGAL::Polygon_mesh_processing::triangulate_faces()
处理对象：输入多边形网格的所有面（无论原始面是三角形、四边形或其他多边形）。
技术步骤：
支撑平面计算：为每个面选择一个近似支撑平面，其法向量通过 CGAL::Polygon_mesh_processing::compute_face_normal() 计算。
约束 Delaunay 三角化：在每个面的支撑平面内，使用 二维约束 Delaunay 三角化（Constrained_Delaunay_triangulation_2） 对该面进行三角化。
优势：约束 Delaunay 三角化可确保在给定面边界边的情况下，生成的三角形最小角度最大化（即三角形形状更规则，避免狭长或退化三角形）。
3. 核心目标与技术价值
目标：通过细化、平滑和三角化，将任意多边形网格转换为满足算法输入要求的高质量三角形网格（如统一面度数、优化局部区域、保证几何规则性）。
应用场景：适用于计算机图形学、几何建模、网格优化等任务，确保后续算法（如渲染、仿真、曲面重建）的稳定性和效率。

Remeshing-Isotropic Incremental Remeshing：
CGAL::Polygon_mesh_processing::isotropic_remeshing()：该 API 可对多边形网格的三角化区域重新进行网格化。它有两个参数，分别是重新网格化后表面片的目标边长和操作序列的迭代次数。迭代次数越多，网格就越光滑且越接近目标边长。
CGAL::Polygon_mesh_processing::split_long_edges()：在重新网格化之前，可调用此 API 对受限边列表进行处理，避免在保护给定多边形线条时出现边分割无限循环的问题。
CGAL::Polygon_mesh_processing::compute_face_normal()：用于计算面的法向量，为CGAL::Polygon_mesh_processing::triangulate_faces()选择支撑平面提供依据。

Remeshing-Surface Remeshing：
算法基础
3D Mesh Generation 包中的网格生成算法可用于重新网格化给定的三角化表面网格。该算法基于受限 Delaunay 三角剖分的 Delaunay 精炼。受限 Delaunay 三角剖分是在普通 Delaunay 三角剖分基础上加入了特定的约束条件，而 Delaunay 精炼则是对生成的三角剖分进行进一步优化。
这种算法生成的三角表面网格在多个方面具有可证明的良好性质，包括简单形（如三角形）的大小、对原始表面的逼近程度、面片（三角形）的形状以及表面的拓扑结构。
约束条件与特征保护
可以将输入网格的一组边作为约束条件。这些约束边用于构建和保护一组多边形特征，并且在重新采样过程中会保持输入特征图的拓扑结构不变。也就是说，在重新网格化时，虽然会对网格进行调整，但会确保指定的特征边和其构成的拓扑关系不被破坏。
相关函数
CGAL::Polygon_mesh_processing::surface_Delaunay_remeshing() 是实现此三角表面重构的函数。它可以直接使用 CGAL::make_mesh_3() 中定义的所有网格划分标准，这些标准涵盖了简单形体的大小、面片形状、表面逼近、拓扑结构以及一维特征采样等方面。
Polygon_mesh_processing/delaunay_remeshing_example.cpp 文件提供了一个示例，展示了如何使用 Delaunay 精炼算法对给定的三角化表面网格进行重新网格化，同时保留检测到的尖锐边缘。

Remeshing-Remeshing of (Almost) Planar Patches：
需求与初步方法
当模型的一个平面区域使用了大量三角形来描述时，为了简化该区域的网格，可以使用较少的元素，甚至在该区域构成简单连通片时使用一个单一的大面片。
CGAL::Polygon_mesh_processing::remesh_planar_patches() 函数可用于实现这一目标。它通过几何判据进行共面性和共线性检查来检测平面区域。然而，由于输入的区域实际上并非完美共面，如果进行精确的共面性和共线性测试，检测到的平面区域可能会意外地小。
容差阈值的问题
为了解决上述问题，可以指定相邻面（或线段）之间的角度阈值，当角度小于该阈值时，它们被视为共面（或共线）。但这个容差阈值是局部的，缺乏全局控制，可能会产生不期望的效果。例如在密集采样的圆弧中，所有点最终可能都被认为几乎共线，导致检测结果不符合预期。
改进方法
CGAL::Polygon_mesh_processing::remesh_almost_planar_patches() 函数解决了局部容差阈值的问题。不过，它需要用户提供平面片和角的分割结果。
CGAL::Polygon_mesh_processing::region_growing_of_planes_on_faces() 函数可用于获取这种分割结果。它使用区域生长算法，结合全局和局部标准来检测网格中的平面区域。
CGAL::Polygon_mesh_processing::detect_corners_of_regions() 函数可以在补丁边界段上运行区域生长算法，检测平面区域边界上的角顶点。
